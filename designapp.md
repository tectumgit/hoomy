Да, фиксируем: **дизайн Хоми отталкивается от логики Яндекс Еды 2026 года** — чистый, современный, мобильный, понятный для Android и iOS.

Важно: **не копируем Яндекс Еду один в один** — это чужой бренд и интерфейс. Берем как UX-референс: структуру, простоту, карточки, навигацию, визуальную чистоту. У Хоми должен быть свой стиль.

# Дизайн-направление Хоми

## Основной референс

**Яндекс Еда / Яндекс Лавка по UX-логике:**

* чистый белый фон;
* крупные карточки;
* понятная типографика;
* минимум лишних элементов;
* быстрый доступ к поиску;
* горизонтальные категории;
* акцент на фото товара;
* простая корзина;
* заметные статусы доставки.

Яндекс Еда сейчас позиционируется как приложение для заказа еды из ресторанов и продуктов из магазинов, включая Лавку, Ленту, ВкусВилл, Пятёрочку, Перекрёсток и другие сети. Это подтверждает, что их UX уже привычен массовому пользователю для еды и продуктов. ([Google Play][1])

---

# Что берем из Яндекс Еды

## 1. Главный экран

Для Хоми главный экран должен быть таким же простым:

* адрес доставки сверху;
* строка поиска;
* горизонтальные категории;
* промо-блоки;
* подборки товаров;
* карточки поставщиков;
* блок «Закупиться на неделю»;
* блок «Оптовые цены»;
* блок «Ближайшая доставка».

## 2. Категории

Вместо ресторанных категорий делаем продуктовые:

* овощи;
* фрукты;
* мясо;
* молочка;
* крупы;
* напитки;
* заморозка;
* бытовая химия;
* фермерское;
* для кафе и бизнеса.

## 3. Карточки

У Яндекс Еды сильная сторона — простые визуальные карточки с фото, рейтингом, временем доставки и понятным действием. Для Хоми карточка должна быть адаптирована под опт.

Пример карточки Хоми:

```text
Картофель фермерский
38 ₽ / кг
Минимум: 10 кг
Шаг: 5 кг
Ближайшая доставка поставщика: 20 мая
Поставщик: Фермерская база №1
[В корзину]
```

## 4. Навигация

Нижняя навигация:

```text
Главная | Каталог | Заказы | Корзина | Профиль
```

Можно сделать так:

```text
Главная | Каталог | Поставщики | Заказы | Профиль
```

Корзину лучше показывать плавающей кнопкой снизу, когда в ней есть товары.

# Что НЕ копируем

Нельзя копировать:

* логотип;
* фирменный желтый цвет Яндекса как основной;
* иконки Яндекса;
* тексты;
* фирменные иллюстрации;
* точную сетку экранов;
* точный вид карточек;
* фирменные паттерны, если они узнаваемы как Яндекс.

Берем только принцип:

**чисто, быстро, понятно, крупно, удобно одной рукой.**

# Визуальный стиль Хоми

## Общая идея

Хоми должен ощущаться как:

**«Яндекс Еда по удобству, но для оптовых закупок продуктов»**

То есть:

* не премиальный сложный интерфейс;
* не маркетплейс с перегрузом;
* не B2B-таблица;
* а простое мобильное приложение для регулярных закупок.

# UI-принципы Хоми

## 1. Белый фон

Основной фон:

```text
#FFFFFF
```

Дополнительный фон блоков:

```text
#F5F6F7
```

## 2. Скругления

Карточки и кнопки:

```text
16–24 px
```

Для мобильного интерфейса это дает современный мягкий вид.

## 3. Крупная типографика

Главные цены должны быть крупными.

Пример:

```text
38 ₽ / кг
```

А не мелким текстом.

## 4. Фото товара — главный элемент

Для еды и продуктов фото продает лучше текста.

Карточка товара должна начинаться с фото, потом цена и условия.

## 5. Минимум текста

Не писать длинно:

```text
Минимальное количество для оформления заказа составляет 10 килограмм
```

Писать коротко:

```text
Мин. 10 кг
```

# Цветовая логика Хоми

Чтобы не копировать Яндекс, лучше выбрать свою палитру.

## Вариант 1 — зеленый

Подходит для продуктов, фермерства, свежести.

```text
Основной цвет: #20A464
Акцент: #FFB020
Фон: #F5F6F7
Текст: #111111
Ошибка: #E53935
Успех: #16A34A
```

## Вариант 2 — теплый оранжевый

Подходит для еды и доставки, но не повторяет Яндекс.

```text
Основной цвет: #FF7A1A
Акцент: #2FA66A
Фон: #F7F7F5
Текст: #111111
Ошибка: #E53935
Успех: #16A34A
```

## Вариант 3 — графит + зеленый

Более современно и серьезно.

```text
Основной цвет: #111827
Акцент: #22C55E
Фон: #F6F7F8
Текст: #111111
Ошибка: #DC2626
Успех: #16A34A
```

## Моя рекомендация

Для Хоми лучше:

```text
Основной цвет: зеленый
Дополнительный акцент: теплый оранжевый
Фон: белый / светло-серый
```

Почему:

* зеленый = продукты, свежесть, экономия;
* оранжевый = акция, цена, выгодно;
* не похоже напрямую на Яндекс Еду.

# Экраны, которые проектируем в стиле Яндекс Еды

## 1. Главный экран

Содержит:

* адрес доставки;
* поиск;
* категории;
* баннер «Закупка на неделю»;
* блок «Популярное оптом»;
* блок «Ближайшая доставка»;
* блок «Поставщики рядом»;
* блок «Семейные наборы».

## 2. Каталог

Содержит:

* поиск;
* фильтры;
* категории;
* сортировка;
* товары карточками;
* цена за кг / л / шт;
* минимальный объем;
* ближайшая дата доставки поставщика.

## 3. Карточка товара

Содержит:

* фото;
* название;
* цена;
* минимальный объем;
* шаг заказа;
* поставщик;
* остаток;
* условия хранения;
* ближайшая дата доставки поставщика;
* кнопка добавления.

## 4. Корзина

Содержит:

* список товаров;
* количество в кг / л / шт;
* минимальный объем;
* сумма;
* доставка;
* дата;
* итог;
* кнопка оплаты.

## 5. Оформление заказа

Содержит:

* адрес;
* дата доставки по выбранному поставщику;
* способ оплаты;
* комментарий;
* итоговая сумма;
* подтверждение.

## 6. Заказы

Содержит:

* текущие заказы;
* статусы;
* дата доставки по каждому поставщику;
* повторить заказ;
* написать в поддержку.

```text
Мин. заказ: 10 кг
```

```text
Шаг: 5 кг
```

```text
от 380 ₽ за 10 кг
```

## 5. Дедлайн заказа

```text
Заказать до 18:00
```

## 6. Плановая доставка

```text
Доставка 20 мая
```

```text
Визуальное направление приложения Хоми должно отталкиваться от современных food delivery приложений уровня Яндекс Еда: чистый мобильный интерфейс, светлый фон, крупные карточки, простой поиск, горизонтальные категории, понятная корзина и быстрый путь к заказу.

Дизайн должен быть самостоятельным и адаптированным под модель оптовых закупок: минимальный объем товара, шаг заказа, цена за кг/л/шт, плановая доставка и поставщики.
```

# Финальное решение

Фиксируем дизайн-направление:

**Хоми = чистота и удобство Яндекс Еды + механика оптовых закупок + собственный бренд.**

То есть:

```text
Референс UX: Яндекс Еда 2026
Дизайн-система: Material 3 + Apple HIG
Фронт: React Native + Expo
Стиль: светлый, чистый, карточный, mobile-first
Не копировать: цвета, логотипы, иконки, тексты Яндекса
Добавить свое: кг/л/шт, минимальный объем, плановая доставка, поставщики
```
