# 6. UX/UI Design System

## Мобильное приложение «Hoomy / Хоми»

---

# 1. Общая идея дизайна

**Hoomy** должен выглядеть как современное приложение доставки еды уровня 2026 года:

* легкий интерфейс;
* крупные карточки;
* много воздуха;
* понятные действия;
* яркий, но не перегруженный стиль;
* акцент на поставщиках;
* простой путь к заказу;
* ощущение надежности и выгоды.

Главный визуальный образ бренда уже задан:
**оранжевый фон, белый логотип, простые иконки продуктов, дружелюбный стиль.**

---

# 2. Позиционирование интерфейса

Hoomy — это не обычный магазин и не экспресс-доставка.

Интерфейс должен объяснять:

```text
Поставщики → каталоги → крупные закупки → плановая доставка
```

Главный принцип:

**пользователь сначала выбирает поставщика, потом товары.**

---

# 3. Стиль интерфейса

## Основной стиль

* современный;
* чистый;
* светлый;
* минималистичный;
* продуктовый;
* дружелюбный;
* mobile-first;
* похожий по удобству на современные приложения доставки еды, но с собственной логикой оптовых закупок.

---

# 4. Визуальная ДНК бренда

## Основные ассоциации

Hoomy должен ассоциироваться с:

* выгодой;
* продуктами;
* закупкой впрок;
* надежными поставщиками;
* простотой заказа;
* домашним запасом;
* свежестью;
* прямой покупкой без лишних посредников.

# 6. Использование цвета

## Основные правила

Оранжевый цвет нельзя использовать везде подряд.
Он должен быть акцентом.

Правильно:

* оранжевая основная кнопка;
* оранжевая цена или скидка;
* оранжевый бейдж;
* оранжевый активный таб.

Неправильно:

* весь интерфейс оранжевый;
* много оранжевых карточек;
* оранжевый текст в длинных описаниях;
* оранжевые второстепенные элементы.

| 16 px  | основной padding          |
| 20 px  | крупные секции            |
| 24 px  | разделение блоков         |
| 32 px  | большие отступы           |

* категории;

# 12. Иллюстрации

Стиль иллюстраций должен продолжать стиль логотипа:

* простые формы;
* яркие цвета;
* продукты;
* коробки;
* пакеты;
* овощи;
* бутылки;
* фермерские элементы;

Лучше использовать:

* плоские 2D-иллюстрации;
* мягкие геометрические формы;
* простые продуктовые символы.

# 18. Карточка товара

## Структура карточки товара

```text
Фото
Название
Поставщик
Цена за единицу
Минимальный объем
Шаг заказа

```

---

## Главное правило

Доставка не отображается как свойство товара.
Доставка относится к поставщику и выбирается при оформлении заказа.

---

# 19. Корзина

Корзина должна быть разделена по поставщикам.

## Структура

```text
Поставщик 1
- товары
- сумма товаров
- минимальная сумма заказа
- доставка от ...
- до минимума осталось ...

Поставщик 2
- товары
- сумма товаров
- минимальная сумма заказа
- доставка от ...

Общий итог
```

---

## Визуальное решение

Можно использовать:

* табы поставщиков;
* карточки поставщиков;
* раскрывающиеся блоки.

Рекомендуется для MVP:

```text
карточки поставщиков одна под другой
```

Так проще и понятнее пользователю.

---

# 20. Оформление заказа

На оформлении клиент выбирает дату доставки.

## Если один поставщик

Пользователь выбирает:

* адрес;
* дату;
* временной интервал;
* оплату.

## Если несколько поставщиков

Пользователь выбирает дату отдельно по каждому поставщику.

---

## Структура блока поставщика на оформлении

```text
Фермер Казань
Товары: 2 800 ₽
Доставка: 300 ₽

Выберите дату доставки
[20 мая] [22 мая] [24 мая]

Время доставки
[10:00–18:00]
```

| Создан             | Серый               | заказ создан         |

# 33. UX-принципы Hoomy

## 1. Сначала поставщик, потом товар

Пользователь должен понимать, у кого он покупает.

---

## 2. Доставка принадлежит поставщику

Не смешивать доставку с товаром.

---

## 3. Минимальный объем всегда виден

Пользователь не должен узнавать о минимуме только в корзине.

---

## 4. Корзина разделена по поставщикам

Пользователь должен видеть разные условия каждого поставщика.

---

## 5. Оформление заказа похоже на Ozon

Сначала корзина, потом выбор даты доставки.

---

## 6. Все действия должны быть очевидны

Основные кнопки:

* Смотреть каталог;
* Добавить в корзину;
* Выбрать дату доставки;
* Оплатить;
* Написать поставщику;
* Открыть спор.

# 35. Итоговое визуальное направление

**Hoomy должен выглядеть как современное, легкое и понятное приложение для закупок у поставщиков.**

Ключевой стиль:

```text
яркий брендовый оранжевый
+
белые карточки
+
мягкие скругления
+
крупные поставщики
+
простые продуктовые иконки
+
понятная корзина по поставщикам
```

Главное правило дизайна:

**не перегружать интерфейс товарами, а помогать пользователю быстро выбрать надежного поставщика и собрать крупный заказ.**
