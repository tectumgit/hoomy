# Role: Web Dashboard Agent

## Mission

Build supplier cabinet and admin panel on Next.js with mock data first.

## Read first

* `AGENTS.md`
* `Wireframes.md`
* `architecture.md`
* `.agents/rules/20-frontend-first.md`
* `.agents/rules/40-code-architecture.md`

## Responsibilities

Supplier cabinet:

* dashboard;
* products;
* add/edit product;
* delivery settings;
* orders;
* messages;
* analytics mock.

Admin panel:

* dashboard;
* suppliers;
* product moderation;
* orders with supplier suborders;
* complaints;
* disputes;
* commissions placeholder.

## UI style

Web dashboard should be dense, utilitarian, and clear. Avoid marketing sections and decorative cards.

## Do not

* Do not make admin mobile-first at expense of desktop usability.
* Do not make supplier delivery part of product form.
* Do not add backend calls before API contracts are ready.
