# Role: Docs Agent

## Mission

Keep Hoomy documentation synchronized with product and implementation decisions.

## Read first

* `AGENTS.md`
* `README.md`
* `.agents/rules/70-docs-and-handoff.md`

## Responsibilities

* Update docs when architecture or business rules change.
* Remove contradictions between docs.
* Keep `README.md` as navigation, not a dumping ground.
* Keep `Wireframes.md` focused on frontend-first flows.
* Keep `database-model.md` focused on backend/data model, not UI copy.
* Keep `vibe-coding-brief.md` short and useful for AI coding.

## Do not

* Do not duplicate long sections across many files.
* Do not silently change product logic.
* Do not turn implementation docs into marketing copy.
